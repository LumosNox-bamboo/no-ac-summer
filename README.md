# 没有空调的夏天

**No AC Summer: Dorm Survival** 是一个纯前端、浏览器可玩的轻量级模拟经营生存小游戏原型。玩家是一名住在欧洲学生宿舍里的留学生，需要在连续热浪中管理室温、体温和 San 值，每天完成吃饭、学习和睡觉三件事。

## 如何运行

直接双击 `index.html`，或在浏览器里打开该文件即可游玩。项目不需要后端、数据库、npm、构建工具或网络连接。

## 文件结构

```text
survivenoACdorm/
├── index.html   # 页面结构和弹窗容器
├── style.css    # 响应式界面、热浪视觉风格、手机布局
├── game.js      # 游戏配置、状态、渲染、行动、天气、存档和结局
└── README.md    # 项目说明
```

## 已实现功能

- 开始界面：选择简单/困难模式、房间朝向、楼层。
- 每天按小时推进：6:00 到 24:00，每小时 1 个主要行动，23:00-24:00 必须睡觉。
- 核心状态：室温、体温、San、学习进度、吃饭进度、睡眠失败连击、生病模式。
- 天气系统：20 个天气事件，影响升温、通风、夜间降温和 San 惩罚。
- 宿舍事件：40 个房间/宿舍随机事件，影响家具行动、道具、学习和睡眠。
- 家具点击系统：窗户、破窗帘、电脑、冰箱、门、床、厨房、衣柜、风扇。
- 每个家具都有真实可执行行动，没有空按钮。
- 睡眠结算：正常睡、地板睡、湿毛巾睡、冰袋抱睡。
- 道具系统：冰块、旧毛巾、夹子、喷雾瓶、冰袋、耳塞、遮阳布、凉席、二手风扇。
- 生病模式：体温过高后进入，恢复困难，持续不恢复会失败。
- 胜利/失败结局：坚持完整热浪胜利，并按状态给出不同结局。
- `localStorage` 存档：支持继续游戏、重置游戏。
- 响应式布局：电脑和手机竖屏都可以点击游玩。
- 新手指导浮层：进入游戏后显示，可关闭并随存档保存。

## 数值系统说明

游戏刻意不做金钱、商店、饥饿值、能量值等复杂系统，只保留三条最重要压力线：

- `roomTemp`：室温，受天气、朝向、楼层、窗帘、门窗、家具行动影响。
- `bodyTemp`：体温，受室温、San、生病模式、冰块、风扇、冷水澡和睡眠方式影响。
- `san`：精神/体力/忍耐力合并值，受高温、睡眠、学习、吃饭和随机事件影响。

每日必须完成：

1. 吃饭达到当天进度（热饭 +2，冷食 +1，零食 +0.5）
2. 学习达到当天需求（简单 6 小时，困难 8 小时；每次学习最多 +1）
3. 睡觉

睡前吃饭进度不足或学习不足会扣 San。连续睡不好、体温过高或 San 归零都会失败。

## 如何调整 GAME_CONFIG

所有主要平衡参数都集中在 `game.js` 顶部的 `GAME_CONFIG`：

- `modes`：简单/困难模式参数。
- `orientations`：朝北、朝东、朝南、朝西的升温和直射规则。
- `floors`：低楼层、中楼层、顶楼的差异。
- `phases`：6:00 到 24:00 的小时推进表。
- `thresholds`：失败阈值、学习需求、日志上限。
- `weatherEvents`：天气事件池。
- `dormEvents`：宿舍/房间事件池。
- `furniture`：家具显示信息。

如果要调难度，优先改：

- `initialSan`
- `initialRoomTemp`
- `heatwaveMin / heatwaveMax`
- `thresholds.bodyFail`
- 天气和事件的 `effects`

## 本轮调参说明

- 室外最低温调整为 20-25°C，最高温调整为 33-38°C。
- 每天按小时推进，非睡眠小时都有 1 个主要行动。
- 开窗/关窗/夜间大通风不消耗小时，但每小时只能调整一次窗户。
- 普通开窗不主动降温；只有短时大风、21:00 后大风、夜间冷空气或风扇对窗外排热时，开窗才可能降低室温。
- 今日天气面板会公布房间朝向对应的阳光直射时间。
- 学习需求提高到 6-8 小时，且每次学习最多 +1；刷手机不学习，但可以回复 San。
- 吃饭需求提高，冷食和零食通常需要多次搭配才达标。
- 简单模式开局固定获得二手风扇 +1，并额外获得 1 个基础小道具；困难模式不保底。

- 21:00 才算入夜，21:00-24:00 是快速降温窗口。
- 门、窗、窗帘状态会在每小时结算中持续影响室温或 San。
